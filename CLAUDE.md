# Контекст проекту для Claude

## Що це

Тренажер для підготовки до українського ліцензійного іспиту КРОК-2 Стоматологія. Імітує реальний іспит: 150 запитань × 200 хв, прохідний бал 64%.

## Поточний стан (v0.1.0-seed)

- ✅ Веб-додаток на Alpine.js у `docs/` (SPA: старт → іспит → результати)
- ✅ Сидові запитання по ~10 на кожен з 7 розділів у `docs/questions/`
- ✅ GitHub Actions workflow для деплою
- ⏳ Пайплайн генерації повної бази (`scripts/`) — заплановано

## Архітектура

### Веб-сайт (`docs/`)
- Один HTML + Alpine.js (з CDN) + vanilla CSS
- 7 JSON-файлів запитань, по одному на розділ
- Завантажуються паралельно при `init()` через Promise.all
- Weighted random sampling 150 запитань за пропорціями з `config.json`
- Stateless — нічого не зберігається, як просив користувач

### Пайплайн (`scripts/`)
- Python з `uv`, залежності в `scripts/pyproject.toml`
- Anthropic SDK з Batch API + prompt caching
- Промпти у `scripts/prompts/rewrite.md` та `scripts/prompts/classify.md`
- Модель за замовчуванням: `claude-haiku-4-5`, валідація через `claude-sonnet-4-6`

```json
  "explanation_ua": "..."

```

## Ключові рішення користувача

- Джерело: ручний парсинг публічних джерел + переформулювання через Claude API
- Точність: «сбалансовано» — Claude змінює деталі пацієнта, зберігає клінічну концепцію та правильну відповідь
- Без зображень (у v1)
- Без збереження прогресу (stateless)
- Тільки повний режим 150 запитань (не мінітести в v1)

## Що пам'ятати

- Мова **виключно українська** (для UI і для запитань)
- Цільова аудиторія — **5-й курс стоматологічного факультету**
- Сидові запитання у `docs/questions/*.json` — для тестування UI, замінити при генерації повної бази
- Пропорції в `config.json` — стартові; уточнити після класифікації спарсених даних
